✅ CHECKLIST ULTRA RIGOROSO – 32 ITENS
1. Padrão de Design (UI / UX / Design System)

Cores, fontes, grid, espaçamentos, componentes, estados visuais, responsividade.
Mapeamento: Figma → Tailwind → Vuetify.

2. Infraestrutura

APIs, filas, cache, banco, storage, serviços de IA, resiliência, tolerância a falhas.

3. Segurança

Autenticação, autorização, proteção de dados, XSS, CSRF, SQL Injection, IDOR, brute force.

4. Backend e Frontend (Contratos)

Endpoints, payloads, status codes, versionamento, responsabilidades.

5. Banco de Dados

Modelagem, integridade, índices, performance, concorrência.

6. Integração Interna com Outros Módulos

Módulos afetados, eventos, dependências ocultas, acoplamento.

7. Arquitetura Geral

DDD, camadas, microserviços, monólito modular, agentes de IA.

8. Sincronização entre Back, Front, Dados, IA e Serviços

Fluxo end-to-end, transações, consistência eventual, retries.

9. Uso de Dados Mockados

Ambiente de desenvolvimento x produção.

10. Testes Automatizados

Unitário, integração, E2E, cobertura mínima.

11. Documentação Básica da Feature

O que é, para quem, onde se encaixa.

12. Performance

Gargalos, cache, filas, latência, volume.

13. Acessibilidade

Semântica, contraste, navegação por teclado, ARIA.

14. Revisão de Código

Clean Code, DRY, KISS, SOLID.

15. Estrutura Modular

Separação de responsabilidades, componentes reutilizáveis.

16. Documentação Detalhada para Tarefas

Histórias, regras de negócio, cálculos, validações.

17. Cards de Especificação (Estilo Jira)

Título, descrição, história, regras, cálculos, riscos, DoD, AC.

18. Roadmap em Formato de Fluxos

Fases, dependências, marcos.

19. Diagrama da Jornada da Feature

Visão do usuário e visão do sistema.

20. Sequência Lógica de Desenvolvimento

MVP, incrementos, redução de risco.

21. Priorização no Produto

Valor x Complexidade x Risco.

22. Tempo Estimado de Desenvolvimento

Ideal, realista, pessimista.

23. Documento de Padrão de Design Consolidado

Tokens, classes Tailwind, componentes Vuetify.

24. Direcionamento Técnico para Rails + Vue + Nuxt + Vuetify

Controllers, services, models, componentes Vue, rotas, store.

25. Integrações via API, Webhook ou MCP

Endpoints, autenticação, retries, idempotência.

26. Uso de Inteligência Artificial

Entrada, saída, modelo, riscos, fallback, custos.

27. Separação Back x Front nos Fluxos

Responsabilidades explícitas por etapa.

28. Consistência Tailwind ↔ Vuetify

Mapeamento automático de componentes.

29. Requisitos Funcionais

O que o sistema deve fazer.

30. Requisitos Não Funcionais

Performance, segurança, escalabilidade, auditoria.

31. Riscos e Recomendações Finais
