Tópicos de análise comparativo – Replit 

 

 

Banco de Dados: 

A estrutura de banco de dados é essencial para determinar o que será ou não viável. No Replit não tem banco de dados funcional, nem mesmo com arquivos json. Sendo assim, todas as consultas, pesquisas, atualizações, criações e diversas outras ações envolvendo dados precisam passar por revisão detalhada para entender o que já existe, existirá ou não existirá no sistema, principalmente considerando o que deve conter no MVP. O banco de dados que temos hoje já deve abordar boa parte do conteúdo do Replit.  

Esse ponto é importante, pois todo o código teria que ser reestruturado para usar conteúdo da API, definindo rotas, tratamento de dados, pontos relacionados a desempenho, etc. 

 

 

API/Back-end: 

O Replit não possui rotas de API reais implementadas. Ou seja, todo o conteúdo é montado de maneira manual pelo front-end. Sendo assim, teríamos que dedicar tempo reescrevendo o código para que o conteúdo venha da API e consiga funcionar de maneira viável com a nossa estrutura de banco de dados. 

 

Front-end: Podemos reaproveitar boa parte do código do front, mas sem saber o quão otimizado, componentizado, seguro, estável ele está. Seria necessário um tempo considerável entendendo esses pontos que podem ou não necessitar de revisões. O mais complexo seria implementar conexões com a API em todos os pontos necessários do sistema, fazendo com que tudo funcione conforme o esperado, como por exemplo: Workspaces, paginações, ordenações, etc. 

 

IA: O agente de IA não realiza buscas reais, roteamentos, etc, pois não há um banco de dados ou uma API funcional no projeto. Aparentemente ele apenas responde com regras configuradas em arquivos como lia-agent-system/app/agents/conversation.py. O Python possui models e “controllers”, porém não consegui ver um uso efetivo deles pelo sistema. O modelo de interface de LLM de utilizado é o FastAPI, mas como não usa rotas e nem dados reais é difícil saber se podemos reaproveitar algo. Já estamos usando a arquitetura baseada em nós no nosso modelo v3;
