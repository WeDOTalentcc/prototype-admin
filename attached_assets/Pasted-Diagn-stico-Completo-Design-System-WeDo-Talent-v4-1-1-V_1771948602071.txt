Light-first com dark mode completo
Paleta monocromática (gray-900 base) com acentos coloridos limitados a 10%

6. INCONSISTÊNCIAS E PROBLEMAS IDENTIFICADOS
6.1 Inconsistências Internas
#	Problema	Severidade	Localização
1	Ghost button font-weight 500 enquanto todas as outras variantes usam 600	Baixa	Linha ~1225 e ~2907
2	Touch targets abaixo de 44px — XS(28px), SM(32px), MD(36px) não atendem WCAG mínimo de 44px para mobile	Média	Seção 2.1.2
3	Seção 4.6 numeração duplicada — "4.6 Mapeamento" e "4.6 Tabela de Migração" na mesma numeração; subseções usam "4.5.1" a "4.5.4" dentro da 4.6	Baixa	Linhas 3887-3931
4	VChip rounded="sm" no Vuetify defaults, mas badges usam rounded-full no DS — inconsistência de border-radius para chips	Baixa	Linha 3112 vs seção 2.4
5	Dark mode incompleto em componentes 2.11-2.29 — apenas 2.1-2.10 têm exemplos dark explícitos	Média	Componentes novos
6.2 Inconsistência Ghost Button — Análise
O ghost button usa font-medium (500) enquanto os outros 4 variantes usam font-semibold (600). Isso é provavelmente intencional — ghost buttons são menos enfáticos visualmente, e o peso menor reforça essa hierarquia. Porém não está documentado como decisão deliberada.

6.3 Touch Targets — Análise
Os tamanhos XS a MD ficam abaixo dos 44px recomendados pelo WCAG 2.1 SC 2.5.5 para alvos touch. Porém, a plataforma LIA é desktop-first (recrutadores em computadores), então isso é aceitável com ressalva:

Em versões mobile futuras, tamanho mínimo deve ser MD (36px) com padding touch de 44px
O documento deveria incluir essa nota contextual

#	Lacuna	Impacto	Detalhe
1	Padrões de Chat	Alto — chat é a interface principal	Nenhum padrão de bolha de mensagem, input de chat, typing indicator, thread, markdown rendering, ou sugestões de ação rápida
2	tailwind.config.js de referência	Alto — devs React não têm config unificada	Cores custom, fonts, extend de spacing/radius não estão mapeados para o Tailwind config
3	Integração shadcn/ui	Alto — stack atual é shadcn/ui	Nenhuma orientação sobre quais componentes shadcn usar, customizações necessárias, ou como mapear tokens do DS para o theme do shadcn
