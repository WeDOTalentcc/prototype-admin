CARD PUB-001: Página Pública de Chat Web para Triagem
**Épico:** É5 — Triagem WhatsApp / Web



Titulo: "[FRONT+BACK] Página Pública de Chat Web para Triagem WSI"
Tipo: Feature
Area: Full-Stack
Sprint: 2
Pontos: 13
Prioridade: Critica (P0)
Epic: EPIC-TRIAGEM
Status: A Fazer
Dependencias: TRI-001, AGT-007
Descricao: |
  Página web pública acessível por link único, onde o candidato realiza a triagem WSI
  via chat conversacional com a LIA. Esta é a interface principal de triagem para
  candidatos que não usam WhatsApp ou quando o canal web é preferido.
  BLOQUEANTE CRÍTICO — sem esta página, não há como realizar triagem via web.
Historia de Usuario: |
  Como candidato, eu quero acessar um link recebido por email
  para responder à triagem da vaga conversando com a LIA em um chat web,
  sem precisar criar conta ou instalar nada.
  Como recrutador, eu quero que candidatos sem WhatsApp possam
  ser triados via uma página web com a mesma qualidade da triagem por WhatsApp.
Regras de Negocio:
  1. URL pública com token único por candidato+vaga (ex: /triagem/{token})
  2. Token expira após 7 dias ou após conclusão da triagem
  3. Sem necessidade de login ou cadastro do candidato
  4. Chat conversacional com LIA usando mesmo fluxo WSI do WhatsApp
  5. Responsiva (mobile-first) — maioria acessa por celular
  6. Sessão persistente — candidato pode sair e voltar pelo mesmo link
  7. Indicador de progresso (etapa atual / total de blocos WSI)
  8. Mensagem de boas-vindas personalizada com nome da empresa e vaga
  9. Ao finalizar, exibir mensagem de agradecimento e próximos passos
  10. Rate limiting: máximo 3 sessões simultâneas por token
Requisitos Tecnicos:
  Frontend:
    - Componente ChatTriagem com interface conversacional
    - Layout responsivo mobile-first (min 320px)
    - WebSocket ou SSE para comunicação em tempo real
    - Persistência de sessão via localStorage + token
    - Indicador de digitação (LIA está escrevendo...)
    - Suporte a mensagens de texto, múltipla escolha e escala Likert
    - Tema visual com branding da empresa (logo, cores)
    - Acessibilidade WCAG 2.1 AA
  Backend:
    - Endpoint GET /api/v1/triagem/{token} — validar token e retornar config
    - Endpoint POST /api/v1/triagem/{token}/message — enviar mensagem
    - Endpoint GET /api/v1/triagem/{token}/history — recuperar histórico
    - WebSocket /ws/triagem/{token} — comunicação em tempo real
    - Integração com AgentEntrevistadorWSI para conduzir triagem
    - Geração de token único (UUID v4) com expiração
  Dados:
    - triagem_sessions: token, candidate_id, job_id, company_id, status, started_at, completed_at, expires_at
    - triagem_messages: session_id, role (lia/candidato), content, timestamp, metadata
Integracoes Externas:
  Nenhuma direta — usa infraestrutura interna (LIA Agent System)
Design & Componentes:
  - ChatContainer: container principal do chat (max-width 640px, centralizado)
  - MessageBubble: bolha de mensagem (LIA à esquerda, candidato à direita)
  - InputBar: campo de texto + botão enviar (fixo no bottom)
  - ProgressBar: barra de progresso WSI no topo
  - WelcomeCard: card inicial com logo empresa + info da vaga
  - CompletionCard: card final com agradecimento + próximos passos
  - TypingIndicator: animação "LIA está digitando..."
  - MultipleChoiceCard: card com opções para perguntas de múltipla escolha
Comportamento UI:
  - Auto-scroll para última mensagem
  - Debounce no envio (300ms) para evitar duplicatas
  - Skeleton loading enquanto carrega histórico
  - Fallback offline: "Verifique sua conexão e tente novamente"
  - Tempo máximo de resposta: 30s timeout com retry automático
DoD:
  - [ ] Página acessível por URL pública com token
  - [ ] Chat conversacional funcional com LIA
  - [ ] Responsiva em mobile (320px+)
  - [ ] Sessão persistente (sair e voltar)
  - [ ] Indicador de progresso WSI
  - [ ] Mensagem de boas-vindas personalizada
  - [ ] Mensagem de conclusão com próximos passos
  - [ ] Token com expiração funcional
  - [ ] Testes e2e cobrindo fluxo completo
Criterios de Aceitacao:
  - Candidato acessa link, vê boas-vindas e inicia chat
  - Chat conduz triagem WSI completa (7 blocos)
  - Candidato pode sair e voltar pela mesma URL
  - Após conclusão, recrutador vê resultado no pipeline
  - Funciona em Chrome, Safari, Firefox mobile
