5. LEGACY VS REACT — DUAS GERAÇÕES DE AGENTES
35.1 Visão Geral da Migração
A plataforma está em migração da arquitetura Legacy para ReAct. Ambas coexistem controladas por feature flag:

USE_REACT_AGENTS = True   # Orchestrator usa ReactAgentRegistry
USE_REACT_AGENTS = False  # Orchestrator usa DomainPrompt.process_intent()
Fallback automático: Se ReAct falha com exceção → tenta agente legacy. Se domínio não tem ReAct → usa legacy automaticamente.

35.2 Comparação Técnica
Aspecto	Legacy (18 agentes)	ReAct (7 agentes)
Motor	DomainPrompt.process_intent()	ReActLoop (Thought→Action→Observe)
Decisão de ferramenta	Código decide (if/else hardcoded)	IA decide (raciocina pela descrição)
Chamadas LLM/interação	1 (fixa)	1–5 (dinâmica, máx. 5 iterações)
Tokens/interação	~1.000–3.000	~3.000–15.000
Custo por resultado	~$0.04 (4 trocas)	~$0.04 (1 interação completa)
Flexibilidade	Baixa (rígido)	Alta (autônomo)
Manutenção	Estruturas diferentes entre si	Padrão único de 4 arquivos
O custo por RESULTADO tende a ser similar. ReAct resolve em 1 interação o que Legacy precisaria de 3–4 trocas.

35.5 Anatomy do System Prompt ReAct — 10 Seções Obrigatórias
Todo agente ReAct tem system prompt com estas seções na ordem:

=== IDENTIDADE ===           → Nome, personalidade, tom, idioma
=== FILOSOFIA CENTRAL ===    → Chat como interface principal
=== INSTRUCOES REACT ===     → Como raciocinar (Thought/Action/Observe)
=== ESTAGIOS ===             → Estágios do domínio com campos de cada um
=== COMPLIANCE E ETICA ===   → LGPD, FairnessGuard, regras de validação
=== EXEMPLOS ===             → Few-shot: entrada → raciocínio → resposta
=== CONTRA-ARGUMENTACAO ===  → Quando e como discordar do recrutador com dados
=== CALIBRACAO ===           → Adaptar ao porte (startup/PME/corporação)
=== CONFIRMACOES ===         → Palavras de confirmação/negação em PT-BR
=== REGRAS CRITICAS ===      → Lista de NUNCA/SEMPRE
Calibração por porte:

STARTUP (<50 func.): requisitos flexíveis, equity aceito, velocidade sobre processo
PME (50–500): equilíbrio requisitos/realidade
CORPORAÇÃO (>500): requisitos detalhados, compliance rigoroso
