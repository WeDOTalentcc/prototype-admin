#9 - Enrich Audit Playbook with Architecture, Compliance & Capabilities
What & Why
The self-contained audit playbook at .local/tasks/lia-deep-audit-playbook.md (970 lines, Task #8) covers governance foundations, FairnessGuard, LGPD basics, screening compliance, and agent governance. However, analysis of 5 additional reference documents (~21,000 lines total) revealed ~20 major sections that must be embedded to make the playbook truly comprehensive. Without these, an auditor using only the playbook would miss critical architecture patterns, compliance gaps, legislative requirements, and capability audits.

Done looks like
The playbook .local/tasks/lia-deep-audit-playbook.md is updated with all sections below embedded directly (no external references needed)
The document remains self-contained — any agent can run a complete audit using only this file
All new sections include concrete verification steps, not just descriptions
The playbook grows to approximately 1800-2200 lines while remaining well-organized
Existing content is preserved; new content is added in appropriate locations within the document structure
Out of scope
Creating separate documents — everything goes into the single playbook file
Implementing any code fixes — this is documentation only
Changing the existing report format (PARTE VIII) — only add new audit dimensions/checks to it
