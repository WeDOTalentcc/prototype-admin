Documentação Completa: Metodologia WSI (WeDoTalent Skill Index)
1. VISÃO GERAL DA ARQUITETURA
O WSI usa 4 frameworks científicos para gerar e avaliar perguntas de screening:

Framework	Propósito	Tipo de Pergunta
CBI (Competency-Based Interviewing)	Avaliar comportamentos passados como preditores de performance futura	Contextual (método STAR)
Dreyfus Model	Classificar nível de domínio técnico (5 estágios)	Autodeclaração + validação
Bloom's Taxonomy	Avaliar nível cognitivo (6 níveis)	Microcase/cenário
Big Five (OCEAN)	Avaliar traços de personalidade	Situacional
2. MAPEAMENTO SENIORIDADE → FRAMEWORKS
Senioridade    → Dreyfus Stage → Bloom Levels
junior         → 2 (Iniciante Avançado) → [1, 2, 3]
pleno          → 3 (Competente)         → [3, 4]
senior         → 4 (Proficiente)        → [4, 5]
lead           → 5 (Especialista)       → [5, 6]
executive      → 5 (Especialista)       → [5, 6]
3. ESTRUTURA DE BLOCOS (Pipeline WSI)
O pipeline gera perguntas organizadas em blocos:

Bloco 2: Perguntas padrão da empresa (do banco de dados, configuráveis por tenant)
Bloco 3: Elegibilidade WSI (disponibilidade, modelo de trabalho, faixa salarial) - template-based
Bloco 4: Avaliação Técnica (CBI + Bloom + Dreyfus) - geradas via LLM + templates
Bloco 5: Análise Situacional e Fit (Big Five + CBI comportamental) - geradas via LLM + templates
Distribuição por formato:

compact: { eligibility: 2, technical: 3, behavioral: 3, total: 8 }
full:    { eligibility: 3, technical: 5, behavioral: 4, total: 12 }
4. PROMPTS DE GERAÇÃO (por framework)
4.1 CBI (Contextual) - 60% das perguntas
Gere UMA pergunta CBI (Competency-Based Interviewing) para avaliar: **{competency.name}**
Nível da vaga: {competency.seniority_level}
Tipo: {competency.type}
Princípio CBI: "Comportamentos passados são os melhores preditores de performance futura."
Estrutura: "Conte sobre uma situação em que [contexto específico]. O que você fez e qual foi o resultado?"
Responda APENAS em JSON:
{
  "framework": "CBI",
  "question_type": "contextual",
  "question_text": "[Pergunta aqui]",
  "expected_signals": ["Contexto claro", "Ação técnica", "Resultado mensurável"],
  "scoring_criteria": {
    "score_5": "Contexto complexo + decisões avançadas + impacto quantificado",
    "score_3": "Contexto claro + ação técnica + resultado visível",
    "score_1": "Contexto vago + ação genérica + sem resultado"
  }
}
Fallback (se LLM falhar):
"Conte sobre uma experiência onde você aplicou {competency.name} em um projeto real. Qual foi o contexto, sua ação e o resultado?"

4.2 Dreyfus (Autodeclaração) - 20% das perguntas
Gere UMA pergunta Dreyfus (autodeclaração) para avaliar: **{competency.name}**
Estrutura: "De 1 a 5, quanto você domina [tecnologia]? Pode citar um projeto recente onde aplicou?"
Combina:
- Autodeclaração (score 1-5)
- Validação contextual (projeto real)
Responda APENAS em JSON com mesma estrutura anterior.
Fallback:
"De 1 a 5, quanto você se considera proficiente em {competency.name}? Pode citar um projeto recente onde aplicou essa competência?"

4.3 Bloom (Microcase) - 15% das perguntas
Gere UMA pergunta tipo microcase (Bloom Level {bloom_level}) para: **{competency.name}**
Nível cognitivo esperado: {"APLICAR" se level 3, "ANALISAR" se level 4, "CRIAR" se level 5+}
Exemplos:
- Level 3 (Aplicar): "Como você implementaria [solução]?"
- Level 4 (Analisar): "Como diagnosticaria [problema]?"
- Level 5 (Criar): "Projete [arquitetura/solução]"
Responda APENAS em JSON.
Fallback:
"Como você abordaria um desafio técnico envolvendo {competency.name}? Descreva sua estratégia de solução."

4.4 Big Five (Situacional) - 5% das perguntas
Gere UMA pergunta Big Five (situacional) para avaliar: **{competency.name}**
Tipo: Comportamental/Cultural
Estrutura: "Como você reage quando...", "Descreva uma situação em que..."
Foco em traços OCEAN:
- Openness: Inovação, aprendizado
- Conscientiousness: Organização, entrega
- Extraversion: Comunicação, liderança
- Agreeableness: Colaboração
- Emotional Stability: Pressão
Responda APENAS em JSON.
Fallback:
"Descreva uma situação recente onde você demonstrou {competency.name}. Como você lidou com o desafio e qual foi o resultado?"

5. BANCO DE PERGUNTAS BIG FIVE (Hardcoded)
O sistema tem um banco de perguntas pré-definidas por traço, cada uma com threshold de score para seleção:

Openness (3 perguntas, thresholds: 60/50/70):

"Descreva uma situação onde você propôs uma solução inovadora..."
"Conte sobre uma vez em que você precisou aprender uma tecnologia completamente nova..."
"Você já questionou um processo estabelecido e sugeriu uma mudança?"
Conscientiousness (3 perguntas, thresholds: 60/50/70):

"Como você organiza e prioriza múltiplas tarefas..."
"Descreva uma situação onde precisou entregar com alta qualidade sob pressão..."
"Conte sobre um projeto de longo prazo que gerenciou do início ao fim..."
Extraversion (3 perguntas, thresholds: 60/50/70):

"Conte sobre uma experiência liderando uma equipe..."
"Descreva uma situação onde precisou influenciar colegas..."
"Como você contribui para manter o moral da equipe?"
Agreeableness (3 perguntas, thresholds: 60/50/40):

"Descreva como lidou com um conflito na equipe..."
"Conte sobre uma vez que ajudou um colega a superar uma dificuldade..."
"Como você reage quando recebe feedback crítico?"
Stability (3 perguntas, thresholds: 60/50/70):

"Como você lida com situações de alta pressão e prazos curtos?"
"Conte sobre uma situação onde um projeto não saiu como planejado..."
"Descreva como mantém o equilíbrio emocional em situações de incerteza..."
Lógica de seleção: Os 3 traços com maior score no perfil Big Five são selecionados. Para cada traço, a primeira pergunta cuja threshold é ≤ ao score do traço é escolhida.

6. TEMPLATES TÉCNICOS POR SENIORIDADE (Hardcoded)
Junior (Bloom 2-3):

"Você já utilizou {skill} em algum projeto? Descreva sua experiência e o que aprendeu."
"Qual foi o maior desafio ao aprender {skill}? Como superou?"
Pleno (Bloom 4-5):

"Descreva um projeto onde utilizou {skill} de forma significativa. Contexto, ação e resultado?"
"Qual foi o maior desafio técnico com {skill}? Como diagnosticou e resolveu?"
"Compare diferentes abordagens relacionadas a {skill}. Quando usaria cada uma?"
Senior (Bloom 5-6):

"Descreva uma arquitetura complexa que projetou usando {skill}. Quais trade-offs considerou?"
"Como decide a melhor abordagem técnica com {skill}? Exemplo de decisão difícil."
"Conte quando mentorou alguém em {skill}. Como estruturou o ensino?"
Lead/Executive (Bloom 6):

"Como definiu padrões técnicos de {skill} para sua equipe/empresa?"
"Qual decisão arquitetural mais impactante com {skill}? Resultados a longo prazo?"
7. SCORING DETERMINÍSTICO (100% sem LLM)
7.1 Fórmula Principal
Score = (0.60 × Autodeclaração) + (0.40 × Contexto) - Penalidade + Bônus
Resultado final: clampado entre 1.0 e 5.0

7.2 Extração de Autodeclaração (regex)
Padrões detectados no texto da resposta:

- Regex: "(\d) de 5", "nota (\d)", "(\d)/5", "nível (\d)"
- Keywords: "expert"→5.0, "avançado"→4.0, "intermediário"→3.0, "básico"→2.0, "nunca usei"→1.0
- Se não encontrado: usa o context_score como fallback
7.3 Cálculo de Contexto (indicadores de texto)
Indicadores de ALTA qualidade (→ score 4.0-5.0):
  "milhões", "percentual", "%", "reduzimos", "aumentamos", "métricas", 
  "kpi", "produção", "escala", "enterprise", "liderança", "arquitetura", 
  "decisão", "impacto", "resultado", "otimização"
Indicadores MÉDIOS (→ score 3.0-3.5):
  "projeto", "equipe", "implementei", "desenvolvi", "trabalhei", 
  "cliente", "api", "sistema", "banco de dados", "deploy"
Indicadores BAIXOS (→ reduz score):
  "curso", "tutorial", "estudando", "aprendendo", "básico", 
  "iniciante", "teoria", "acadêmico"
Lógica exata:

Base = 3.0
Se ≥3 indicadores high → 5.0
Se ≥1 indicador high → 4.0 + (count × 0.2)
Se ≥3 indicadores medium → 3.5
Se ≥1 indicador medium → 3.0 + (count × 0.1)
Se ≥2 indicadores low → score - 1.0 (mín 1.0)
Se ≥1 indicador low → score - 0.5 (mín 1.5)
Boost por evidências: +0.1 por evidência (máx +0.5)
7.4 Classificação Bloom (keywords no texto)
Level 6 (Criar): "crio", "arquiteto", "inovo", "lidero", "fundei", "desenvolvi do zero"
Level 5 (Avaliar): "avalio", "julgo", "recomendo", "decido", "valido", "escolhi"
Level 4 (Analisar): "analiso", "comparo", "diferencio", "diagnostico", "otimizo"
Level 3 (Aplicar): "aplico", "uso", "implemento", "desenvolvo", "construo", "faço"
Level 2 (Compreender): "entendo", "explico", "compreendo", "descrevo"
Level 1 (Lembrar): "lembro", "sei", "conheço", "aprendi", "estudei"
Busca top-down (do 6 ao 1), pega o maior nível encontrado.

7.5 Classificação Dreyfus (anos + contexto)
Level 1 (Novato): 0-1 anos
Level 2 (Iniciante Avançado): 1-2 anos
Level 3 (Competente): 2-3 anos
Level 4 (Proficiente): 3-5 anos
Level 5 (Especialista): 5+ anos
Ajustes:
- Se context_score ≥ 4.5 → level + 1 (máx 5)
- Se context_score < 2.5 → level - 1 (mín 1)
Extração de anos (regex):

- "(\d+) anos de experiência"
- "há (\d+) anos"
- "(\d+) anos trabalhando/desenvolvendo"
- Default se não encontrado: 2.0 anos
7.6 Penalidades
Inflação (-1.0): Autodeclaração ≥ 4.5 E contexto < 3.0
  Keywords: "expert", "especialista", "domino completamente", "5 de 5"
Genérico (-0.5): ≥2 keywords genéricas com < 50 palavras
  Keywords: "trabalhei com isso", "tenho experiência", "já fiz", "sei fazer"
Sem contexto (-0.3): Resposta com < 20 palavras
7.7 Bônus
Humildade (+0.5): 
  Keywords: "ainda estou aprendendo", "preciso melhorar", "3 de 5", "intermediário"
Evidência excepcional (+0.3):
  Keywords: "open source", "contribuí", "palestrei", "publiquei", "patente", "prêmio"
Bônus máximo total: 1.0
8. SCORE WSI FINAL (Candidato)
WSI Geral = (0.70 × WSI Técnico) + (0.30 × WSI Comportamental)
Onde:
WSI Técnico = média ponderada dos scores das respostas técnicas
WSI Comportamental = média ponderada dos scores das respostas comportamentais
8.1 Classificação
≥ 4.5: Excelente
4.0-4.4: Alto
3.0-3.9: Médio
2.0-2.9: Regular
< 2.0: Baixo
8.2 Decisão Automática (Cutoffs)
≥ 4.2: Aprovado automaticamente
3.8-4.1: Requer revisão manual
3.0-3.7: Aguardando comparação com outros candidatos
< 3.0: Rejeitado (abaixo do corte mínimo)
9. DEDUPLICAÇÃO
O pipeline usa SequenceMatcher para evitar perguntas duplicadas entre as da empresa e as WSI:

threshold = 0.65 (65% de similaridade textual)
Se similarity ≥ 0.65 → pergunta é descartada
10. REBALANCEAMENTO
Se o pipeline não gerou perguntas suficientes para atingir o target:

Primeiro tenta gerar mais perguntas comportamentais
Se ainda faltam, tenta mais perguntas técnicas
Por último, tenta mais perguntas de elegibilidade
Se ainda não atingiu, emite quality_warning
