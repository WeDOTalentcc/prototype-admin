Sistema --lia-* (LIA Core)
São os tokens "originais" do Design System, definidos em design-tokens.css. Usados para a UI funcional padrão — layout, textos, bordas, backgrounds base.

Exemplo real de uso (em qualquer componente genérico da plataforma):

<div style={{ 
  background: 'var(--lia-bg-primary)',      // #FFFFFF — fundo branco
  color: 'var(--lia-text-primary)',          // #111827 — texto escuro
  borderBottom: '1px solid var(--lia-border-subtle)'  // #E5E7EB — borda sutil
}}>
  <h2 style={{ color: 'var(--lia-text-primary)' }}>Título da Seção</h2>
  <p style={{ color: 'var(--lia-text-secondary)' }}>Descrição em cinza mais claro</p>
</div>
Onde aparece: Sidebar, headers, tabelas, formulários, backgrounds de página. É o "esqueleto" cinza/branco da plataforma — a parte que segue a regra 90/10.

Total de usos no código: ~473 vezes.

Sistema --eleven-* (ElevenLabs)
São tokens inspirados na estética do ElevenLabs — superfícies com mais personalidade visual. Definidos em globals.css. Usados nos componentes de chat, Kanban, cards coloridos, painéis expandidos.

Exemplo real de uso (no chat ou num card de candidato):

<div style={{ 
  background: 'var(--eleven-bg-card)',       // #FFFFFF — card ElevenLabs
  color: 'var(--eleven-text-primary)',       // #2D2D2D — texto um pouco diferente
  border: '1px solid var(--eleven-border-subtle)'  // #E8E8E8 — borda própria
}}>
  <span style={{ color: 'var(--eleven-text-tertiary)' }}>10:32 AM</span>
  <p style={{ color: 'var(--eleven-text-primary)' }}>Mensagem da LIA no chat</p>
</div>
<!-- Badge no Kanban com cor sepia -->
<span style={{ 
  background: 'var(--eleven-sepia-mint)',    // #DCE4DB — verde menta suave
  color: 'var(--eleven-text-primary)' 
}}>
  Aprovado
</span>
Onde aparece: O chat expandido (expanded-chat-modal.tsx), mensagens do chat, Kanban cards, badges pastéis, painéis de IA. São as partes da UI que têm "personalidade" — as cores sepia, os pastéis suaves, os backgrounds diferenciados.

Total de usos no código: ~2.190 vezes — é o sistema mais usado na plataforma.

A diferença visual
Aspecto	--lia-*	--eleven-*
Texto principal	#111827 (gray-900)	#2D2D2D (mais escuro, quase preto)
Texto secundário	#4B5563 (gray-600)	#666666 (cinza puro, não Tailwind)
Background	#FFFFFF / #F9FAFB (Tailwind grays)	#F8F8F8 / #F5F5F5 (cinzas custom)
Bordas	#E5E7EB (gray-200)	#E8E8E8 (cinza puro)
Personalidade	Neutro, funcional, Tailwind puro	Mais quente, tons sepia/pastel, identidade própria
O sistema --eleven-* não usa os grays do Tailwind — ele tem sua própria paleta de cinzas puros (#2D2D2D, #666666, #999999, #E8E8E8, #F5F5F5, #F8F8F8) que dá um look distinto ao chat e aos cards. É o que dá à plataforma aquela sensação "ElevenLabs" nos componentes mais interativos.

Na prática, quando você abre a plataforma: a sidebar, header e tabelas usam --lia-*, e o chat, Kanban cards e painéis da LIA usam --eleven-*.

Não recomendo migrar tudo para --eleven-*. Vou explicar o raciocínio:

Por que NÃO unificar tudo em --eleven-*
Os dois sistemas servem propósitos diferentes. O --lia-* usa a escala padrão do Tailwind (gray-50 a gray-950). Isso é intencional — componentes como sidebar, tabelas, formulários e headers são UI funcional que precisa seguir um padrão previsível e consistente com o ecossistema Tailwind/shadcn. Se você trocar var(--lia-text-primary) (#111827 / gray-900) por var(--eleven-text-primary) (#2D2D2D), vai sair do grid do Tailwind e perder a compatibilidade com as utilities (text-gray-900, bg-gray-50, etc.).

O --eleven-* usa cinzas puros (#2D2D2D, #666666, #E8E8E8) que não existem na escala Tailwind. Funcionam bem para os componentes de chat e Kanban porque esses componentes já estão fora do sistema Tailwind de classes — usam style={} inline. Mas forçar isso nos componentes que usam className="text-gray-900" criaria inconsistência.

A migração para Vue+Vuetify ficaria mais difícil. O Vuetify usa Material Design grays que se alinham mais com o Tailwind gray do que com os cinzas puros do ElevenLabs. Manter o --lia-* como base facilita o mapeamento.

O que EU recomendo
Manter os dois sistemas, mas com regras claras de quando usar cada um:

Contexto	Sistema	Motivo
Layout (sidebar, header, topbar)	--lia-* + Tailwind classes	Funcional, previsível
Formulários, tabelas, modais	--lia-* + Tailwind classes	Compatível com shadcn/ui
Chat e mensagens	--eleven-*	Personalidade, identidade ElevenLabs
Kanban cards e badges	--eleven-* (sepia/pastel)	Visual diferenciado
Painéis expandidos da LIA	--eleven-*	Superfície premium
Componentes novos genéricos	--lia-* + Tailwind classes	Padrão
A única coisa que vale considerar é: no DS, documentar explicitamente essa divisão (que já fizemos na seção 1.2.6) e talvez criar uma regra simples tipo:

"Se o componente faz parte do chat, Kanban ou painéis de IA, use --eleven-*. Para todo o resto, use classes Tailwind (text-gray-900, bg-white) ou tokens --lia-*."
