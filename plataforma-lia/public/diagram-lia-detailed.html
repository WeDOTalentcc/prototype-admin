DATA FLOW — Complete Request Lifecycle
1. User types in Chat → POST /api/v1/chat or /ws/agent-chat with { message, session_id, domain, context }
2. MainOrchestrator → TenantBudget check → CascadedRouter (FastRouter → IntentRouter → LLM) → domain selected
3. PolicyEngine checks hiring policy → FairnessGuard pre-check → PIIMasking strips PII from prompt
4. DomainWorkflow 7-step pipeline: resolve refs → smart extract → analyze intent → execute action → format response
5. Domain agent (ReAct / LangGraph / Multi-Agent) executes with tool calls → AuditCallback logs everything
6. SSE/WebSocket stream back → Frontend renders progressively → Prometheus metrics recorded → user sees result
