WORKFLOW LAYER — Two Coexisting LangGraph Pipelines
WorkflowOrchestrator (src/workflow/graph.py — 13KB) — Multi-Agent Pipeline
LangGraph StateGraph with QueryState. 6-node pipeline: IntentAnalyzerAgent → APIPlannerAgent → PlanValidatorAgent → APIExecutorAgent → DataProcessorAgent → AnswerFormatterAgent
Agents defined in src/agents/ (intent_analyzer, api_planner, plan_validator, api_executor, data_processor, answer_formatter). Uses src/tools/ (ToolRegistry + ToolExecutor + contracts + hooks).
DomainWorkflow (src/domains/workflow.py — 22KB) — Per-Domain Intent+Execute+Format
LangGraph StateGraph with DomainState. 3-node graph: DomainIntentAgent (LLM classifier → action_id) → Domain.execute_action() → Response Formatting
Entry point for ALL 8 domains. Uses LLMCacheService, CircuitBreaker, PII masking (log only), ResponseFilter (tone).

DATA FLOW — Complete Request Lifecycle
1. User types in Prompt → prompt_input.vue POSTs to /chat/stream with { message, session_id, domain, context_data: { ui_context, current_path } }
2. FastAPI → MessageRouter → HubOrchestrator → HubPlanner (fast-path / CostLadder / LLM) → HubExecutor
3. DomainOrchestrator builds DomainContext (enriches ui_context + viewing_entities) → DomainWorkflow (injects Audit+Streaming+Thinking callbacks)
4. DomainIntentAgent classifies intent → Domain.execute_action() → api_client.py calls Rails API → response
5. SSE stream back → StreamingMessage.vue renders progressively → ExecutionTracker shows steps → user sees result
